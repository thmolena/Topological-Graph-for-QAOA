## TL;DR

QAOA's cost is dominated by the *number of circuit evaluations* spent searching for good
variational angles. This project learns a **relabeling-invariant map from graph topology to good
depth-1 angles**, and uses it as a **warm start** for a query-budgeted local refiner. On
MaxCut over six graph families (the structures that arise in molecular-interaction and
patient-similarity graphs), the topology-conditioned warm start reaches a higher
approximation ratio **and hits the target ratio in far fewer queries** than random or
purely spectral initialization — and it **transfers to graph families never seen during
training**.

Every number in this README is generated by the included code (`make demo`); nothing is hand-edited.

## Method

```
 graph G ──► relabeling-invariant descriptors ──► graph-conditioned policy ──► warm-start (γ,β)
            (degree·motif·WL·cycle·Laplacian)      (kNN over descriptors;            │
                                                    optional torch GNN)              ▼
                                          query-budgeted refiner  ◄──── exact MaxCut reward
                                          (counts every evaluation)     (brute-force oracle)
```

1. **Descriptors** ([`descriptors.py`](code/src/topoqaoa/descriptors.py)) — degree, motif/clustering, hashed
   Weisfeiler–Lehman, cycle, Laplacian-spectrum and connectivity features. Every feature is
   a graph invariant, so the policy is node-ordering invariant *by construction* (verified
   in [`test_descriptor_invariance.py`](code/tests/test_descriptor_invariance.py)).
2. **Objective** ([`qaoa.py`](code/src/topoqaoa/qaoa.py)) — an exact statevector simulator **and** the
   analytic depth-1 MaxCut formula (Wang et al., 2018). The test suite asserts they agree to
   `1e-9` on random graphs from every family, so the O(|E|) closed form that powers the
   in-browser demo is provably correct.
3. **Policies** ([`baselines.py`](code/src/topoqaoa/baselines.py)) — `random`, `spectral`, `topology`
   (fixed-angle heuristic), `graph_conditioned` (the contribution), and `rl` (cross-entropy
   refinement). All compared at a **matched query budget**.
4. **Audited reward** — the exact approximation ratio against a brute-force MaxCut oracle
   ([`maxcut_exact.py`](code/src/topoqaoa/maxcut_exact.py)).

### Library — `code/src/topoqaoa/`, file by file

| File | Contribution |
|---|---|
| [`graph_generators.py`](code/src/topoqaoa/graph_generators.py) | The six benchmark families — Erdős–Rényi, 3-regular, Barabási–Albert, Watts–Strogatz, grid, SBM — plus a random-relabeling utility used to test invariance. |
| [`maxcut_exact.py`](code/src/topoqaoa/maxcut_exact.py) | Brute-force MaxCut oracle (exact for n ≤ 20). Defines "ground truth" for every approximation ratio — the audited reward. |
| [`descriptors.py`](code/src/topoqaoa/descriptors.py) | Relabeling-invariant topology descriptors (degree · motif/clustering · hashed Weisfeiler–Lehman · cycle · Laplacian spectrum · connectivity). **The representation the whole method rests on.** |
| [`qaoa.py`](code/src/topoqaoa/qaoa.py) | Two ways to compute ⟨C⟩: a full statevector simulator and the analytic depth-1 closed form. Cross-checked to 1e-9 — the closed form is what runs in the browser demo. |
| [`env.py`](code/src/topoqaoa/env.py) | Query-budgeted refiner: a coordinate-ascent search that **counts every objective evaluation**, so "queries-to-target" is honest. |
| [`baselines.py`](code/src/topoqaoa/baselines.py) | All five warm-start policies behind one interface — random, spectral, topology (fixed), **graph-conditioned (the contribution)**, and RL. |
| [`splits.py`](code/src/topoqaoa/splits.py) | Family-held-out train/test splits and an explicit leakage check, so transfer claims are real. |
| [`metrics.py`](code/src/topoqaoa/metrics.py) | Approximation ratio, regret, queries-to-target, hit rate, and bootstrapped 95% CIs. |
| [`runner.py`](code/src/topoqaoa/runner.py) | Orchestrates the full protocol end to end and writes `results/summary.json`. |
| [`audit.py`](code/src/topoqaoa/audit.py) | The readiness gate: scans for forbidden superiority claims and verifies every reported number traces to a generated artifact. |
| [`plotting.py`](code/src/topoqaoa/plotting.py) | Renders the query-budget frontier and per-family figures into `figures/`. |
| [`config.py`](code/src/topoqaoa/config.py) · [`seed.py`](code/src/topoqaoa/seed.py) | YAML config loading and deterministic seeding (reproducibility plumbing). |

## How to use

```bash
cd code
make install         # numpy / scipy / networkx / scikit-learn / matplotlib (CPU-only)
make test            # 15 tests incl. closed-form vs statevector cross-check
make demo            # ~1 s: writes results/summary.json
make tables figures  # results/main_results.{tex,md}, figures/*.pdf
make audit           # readiness gate: traceable numbers, no forbidden claims
make full-run        # reported-scale configuration
```

Or open [`index.html`](index.html) directly in any browser to explore the method interactively — it
re-implements the exact MaxCut oracle and the analytic depth-1 QAOA formula in pure JavaScript, so
what you see on the page is the same math as the Python package.

## Highlights — what to look at first

- **[`index.html`](index.html)** — the fastest way to understand the idea: graph → topology → warm-start angles → approximation ratio, live.
- **[`descriptors.py`](code/src/topoqaoa/descriptors.py)** — the invariant representation that makes the policy node-ordering-proof by construction.
- **[`qaoa.py`](code/src/topoqaoa/qaoa.py)** + **[`test_qaoa_closed_form.py`](code/tests/test_qaoa_closed_form.py)** — a fast closed form *proven* against a full simulator.
- **[`audit.py`](code/src/topoqaoa/audit.py)** — the discipline: `make audit` fails the build on any number that doesn't trace to a generated artifact.

## GPU / acceleration note

The reference implementation is **CPU-only and runs everywhere**. The architecture exposes
optional accelerated backends ([`code/requirements-optional-gpu.txt`](code/requirements-optional-gpu.txt)):
a **cuQuantum** statevector/tensor-network path for `n > ~20` qubits, a **PyTorch-Geometric**
message-passing GNN as a drop-in for the descriptor policy, and GPU-batched descriptor
extraction. None are required to reproduce the results.
