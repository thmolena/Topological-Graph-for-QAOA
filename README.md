## Summary

The dominant cost of the Quantum Approximate Optimization Algorithm (QAOA) is the number of
circuit evaluations consumed while searching for variational angles, so a warm start that
supplies a near-optimal angle schedule could remove most of that cost. Learned warm starts,
however, have a reputation for matching — not beating — simple physics-based initializers, a
verdict largely formed at depth one, where a single spectral scale already fixes the only
relevant angle. This artifact revisits the question as a function of depth.

The central object is a **spectral-truncation kernel**: each graph is represented by a finite
low-frequency window of its normalized-Laplacian spectrum (a spectral truncation of the graph
operator, adapting the C\*-algebraic spectral-truncation kernels of Hashimoto et al. from
operators to graphs), giving a relabeling-invariant, positive-definite kernel under which the
single most similar *training* graph donates its optimized depth-`p` schedule. Transferring one
real optimum, rather than averaging several, preserves basin consistency and is near-optimal on
the first query. On a query-counted, family-held-out MaxCut benchmark with an explicit leakage
check, this kernel-transfer warm start **ties an adiabatic ramp at depth one and surpasses it for
`p ≥ 2` by a margin that grows with depth**, attaining the best one-shot ratio at every depth.

Every number in this document is generated by the included code and traces to a committed
artifact (`submission/code/results/summary.json`); none is hand-edited. The accompanying
`index.html` recomputes the depth-`p` QAOA expectation and the exact MaxCut oracle in JavaScript,
so the in-browser demonstration computes the same quantities as the Python package.

## Principal contributions

1. **A depth-resolved finding.** Within a controlled, query-counted, leakage-checked benchmark,
   the depth-one "learned warm starts only match" verdict is shown to be an artifact of `p = 1`:
   topology-conditioned warm starts tie an adiabatic ramp at depth one but surpass it for `p ≥ 2`
   by a margin that grows with depth.
2. **A transferable warm-start operator.** The spectral-truncation kernel — a positive-definite,
   relabeling-invariant kernel on the truncated Laplacian spectrum whose nearest neighbour
   transfers a single optimized depth-`p` schedule. It attains the best final and one-shot
   approximation ratios at the tested depths and is the most query-efficient warm start.
3. **A cross-verified, reproducible benchmark.** A proven relabeling-invariant descriptor and
   kernel, a depth-one objective cross-verified to 1e-9 by analytic and statevector evaluators,
   exact approximation ratios against a brute-force oracle, family-held-out transfer with a
   programmatic leakage check, and a `pip`-installable package that regenerates every figure,
   table, and number from fixed seeds.

## Main results (reported scale)

Generated from `submission/code/results/summary.json`. Scale: 84 graphs (six families, sizes
`n ∈ {8, 10, 12, 14}`), depths `p ∈ {1, 2, 3}`, query budget 28, seed 0. `Δ_final` and
`Δ_one-shot` are the *paired* mean differences between STK transfer and the best per-instance
adiabatic ramp on the final-budget and first-query (`q = 1`) ratios.

| depth `p` | best ramp | descriptor mean | STK transfer (ours) | oracle | Δ_final | Δ_one-shot |
|---|---|---|---|---|---|---|
| 1 | 0.7825 | 0.7824 | 0.7825 | 0.7827 | +0.0000 ± 0.0001 | +0.0030 |
| 2 | 0.8394 | 0.8470 | **0.8503** | 0.8536 | +0.0103 ± 0.0019 | +0.0416 |
| 3 | 0.8622 | 0.8925 | **0.8902** | 0.8999 | +0.0262 ± 0.0026 | +0.0454 |

At depth one the spectral-truncation-kernel transfer and the adiabatic ramp are statistically
indistinguishable (paired difference +0.0000 ± 0.0001). Beyond depth one the parity breaks: STK
exceeds the strongest per-instance ramp by +0.0103 (final) and +0.0416 (one-shot) at `p = 2`, and
by +0.0262 and +0.0454 at `p = 3`, with both `p ≥ 2` final advantages lying outside their paired
95% confidence intervals. The averaging-based descriptor mean also improves on the ramp at depth,
so topology conditioning — not the specific estimator — is what helps; transfer is distinguished
by attaining the best one-shot quality (near-optimal on the first query) at every depth. The
per-family breakdown and the query-budget frontier are written to `summary.json` and plotted in
`submission/code/figures/`.

## Installation

```bash
pip install ./submission/code           # installs the package "topoqaoa"
```

The reference implementation is CPU-only and depends on numpy, scipy, networkx, scikit-learn,
matplotlib, and pyyaml. Optional accelerated backends are declared under the `gpu` extra.

## Reproduction

The package installs a console entry point that deterministically regenerates the tables,
figures, and LaTeX macros from a fixed configuration and seed:

```bash
topoqaoa-reproduce                               # reported scale (configs/full.yaml, seed 0)
topoqaoa-reproduce --config configs/smoke.yaml   # laptop-scale check (seconds)
topoqaoa-reproduce --skip-run                     # tables and figures from an existing summary
```

Equivalently, from `submission/code`:

```bash
python3 scripts/run.py --config configs/full.yaml --out results
python3 scripts/make_tables.py     # writes results/macros.tex and the table bodies
python3 scripts/make_figures.py    # writes the depth, frontier, family, and schematic PDFs
pytest -q                          # closed-form vs statevector (1e-9), invariance (1e-8), kernel PSD, leakage
```

The runner writes `results/summary.json` (the single source of truth) and the figure/table
generators propagate it to the manuscript, which `\input`s the auto-generated macros so that no
number is transcribed by hand.

The browser demonstration requires no installation: open [`index.html`](index.html) in any
browser to select a graph, compute the depth-`p` QAOA approximation ratio of the adiabatic ramp
against a near-optimal schedule (the object the kernel transfers), and watch the gap grow with
depth against the exact-MaxCut oracle.
