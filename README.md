## Summary

The dominant cost of the Quantum Approximate Optimization Algorithm (QAOA) is the number of
circuit evaluations consumed while searching for variational angles. This artifact studies
whether topology conditioning supplies a useful warm start under that constraint. It maps a
graph through relabeling-invariant topology descriptors to depth-1 angles, supplies those
angles to a query-counted local refiner, and measures the approximation ratio against an exact
MaxCut oracle on six graph families. Every warm-start policy is compared at a matched query
budget, the refiner counts each objective evaluation, and the descriptor representation is
invariant to node relabeling by construction. Under this protocol the topology-conditioned
warm start matches spectral initialization; it does not surpass it. The accompanying
`index.html` reimplements the exact MaxCut oracle and the analytic depth-1 QAOA expression in
JavaScript, so the in-browser demonstration computes the same quantities as the Python package.

Every number in this document is generated by the included code and traces to a committed
artifact (`code/results/summary.json`); none is hand-edited.

## Principal contributions

1. **A query-counted MaxCut benchmark.** A coordinate-ascent refiner counts every objective
   evaluation, so queries-to-target and the query-budget frontier are reported on a measured
   rather than nominal basis.
2. **Relabeling-invariant topology descriptors.** Degree, motif and clustering, hashed
   Weisfeiler-Lehman, cycle, Laplacian-spectrum, and connectivity features are graph invariants,
   which renders the conditioned policy node-ordering invariant by construction and is asserted
   in the test suite.
3. **A cross-checked QAOA objective.** A full statevector simulator and the analytic depth-1
   MaxCut expression agree to a tolerance of 1e-9 across every family, so the closed form that
   drives the in-browser demo is verified against simulation.
4. **A matched-budget head-to-head with an honest readout.** Random, spectral, topology,
   graph-conditioned, and a cross-entropy refinement policy are evaluated against an exact
   oracle under family-held-out splits with an explicit leakage check. The topology-conditioned
   warm start matches spectral initialization within confidence intervals and does not surpass
   it.

## Headline results (reported scale)

Transcribed verbatim from `code/results/main_results.md`, generated from
`code/results/summary.json`. Scale: 120 graphs (six families, 20 instances per family, sizes
n in {10, 12, 14, 16}), depth 1, query budget 40, seed 0.

| policy | approx. ratio | ±95% CI | queries→target | hit rate |
|---|---|---|---|---|
| random | 0.7327 | 0.0162 | 40.00 | 0.00 |
| spectral | 0.7846 | 0.0093 | 40.00 | 0.00 |
| topology | 0.7846 | 0.0093 | 40.00 | 0.00 |
| graph conditioned | 0.7846 | 0.0093 | 40.00 | 0.00 |
| rl | 0.7808 | 0.0092 | 40.00 | 0.00 |

At full precision the mean approximation ratios are spectral 0.784553 and graph-conditioned
0.784594, a difference of 4.1e-5 against a 95% confidence half-width near 0.0093. The
graph-conditioned warm start therefore matches spectral initialization to within the confidence
interval and does not surpass it. The target ratio of 0.95 is reached by none of the policies
within the budget, so the target hit rate is 0.00 and queries-to-target equals the budget of 40
for every policy. The per-family breakdown and the query-budget frontier curve are written to
`code/results/summary.json` and plotted in `code/figures/`.

## Installation

```bash
pip install topoqaoa
```

The reference implementation is CPU-only and depends on numpy, scipy, networkx, scikit-learn,
matplotlib, and pyyaml. Optional accelerated backends are declared under the `gpu` extra.

## Reproduction

The package installs a console entry point that deterministically regenerates the tables and
figures from a fixed configuration and seed:

```bash
topoqaoa-reproduce                              # reported scale (configs/full.yaml, seed 0)
topoqaoa-reproduce --config configs/smoke.yaml  # laptop-scale check (a few seconds)
topoqaoa-reproduce --skip-run                    # tables and figures from an existing summary
```

The entry point runs the experiment, writes `results/summary.json`, and emits
`results/main_results.{md,tex}` together with the figure PDFs in `figures/`. Equivalent
Makefile targets are documented in [`code/README.md`](code/README.md).

The browser demonstration requires no installation: open [`index.html`](index.html) in any
browser to generate a graph, observe the topology-conditioned warm start select depth-1 angles,
and compare the resulting approximation ratio against the exact MaxCut optimum.
